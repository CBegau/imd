Compiling IMD

IMD is compiled with GNU make, which we call gmake in the following. Note that on some systems, GNU make is called make. Most other implementations of make won't work.

    gmake [EXTRAFLAGS="some flags"] imd-target
imd-target consists of several components and has the following structure:

    imd_architecture_ensemble[_ensemble]...[_2d][_option]_[option]...

Default is a binary for 3d simulations. For 2d simulations, the component _2d must be specified. In order that processor specific flags are set correctly, the $HOSTTYPE environement variable must be set to one of the supported $HOSTTYPE values. The EXTRAFLAGS variable can be used to define further flags for the compiler, linker or C preprocessor. Note that starting with release 1.3, an IMD binary can support several ensembles. The active one is selected with a parameter read from the parameter file.

Supported architectures:
risc serial execution on one CPU (even on vector computers)
mpi parallel execution using MPI

Supported ensembles:
nve constant volume, constant energy
nvt constant volume, constant temperature, with Nosé-Hoover thermostat
and constant volume, constant temperature, with Anderson thermostat
npt_iso constant pressure, constant temperature, with isotropic volume scaling, Nosé-Hoover thermostat and volume control
npt_axial constant pressure, constant temperature, with volume scaling along each axis independently, Nosé-Hoover thermostat and volume control
mik microconvergence relaxation
mikshear shearing plus microconvergence relaxation
frac fracture simulation with stadium damping
pull ensemble for shearing and pulling
mc Monte Carlo integrator

Supported options:
2d 2d simulation rather than 3d
msqd compute mean square displacements
correl compute correlation functions
eam use EAM potentials
ttbp use TTBP potentials
debug compile with debug flags
prof compile with profiling support
shock generate a shock wave
quasi needed to generate a binary icosahedral quasicrystal (ticq)
disloc dislocations, used for selecting particles in write routines
sock enable sockets communication

Supported values for $HOSTTYPE:
iris4d SGI running Irix 5 or 6
decstation Decstation running Ultrix 4.4
alpha DEC Alpha running Digital Unix 3.2
speedy DEC Alpha running Digital Unix 4.0 (newer compiler)
rs6000 RS6000 with Power2 Architecture running AIX
paragon Intel Paragon running OSF/1
c90 Cray C90
t3e Cray T3E
cray-t3e Cray T3E
HITACHI Hitachi running HI-UX/MPP
i386-linux Linux/Intel
i386-FreeBSD FreeBSD/Intel
os2 IBM OS/2
sx4 NEC SX4
